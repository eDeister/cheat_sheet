Ideal Scenario

In the ideal scenario, edits are made sequentially and by one team member at a time. For example, if someone wanted to make changes to multiple pages, they would first ask if anyone was currently working on the website. Assuming the answer is no, they would then pull the latest changes from the repo using the "git pull" command in their SSH session. Lastly, they download the changes from WHFS, make their changes, and add-commit-push to github. Then repeat for the next student.

Undo changes before git commit

To revert changes before a commit is made, a four-step process must be followed:

  1. Delete the file with the unwanted changes from the local directory (left side of PHPStorm.) This should automatically delete the page from WHFS (right side) but if it didn't, the page can be deleted manually.
  2. Execute a checkout in the terminal: "git checkout -- filename"
  3. Get the latest changes from github using git pull
  4. Refresh the remote host connection in PHPStorm and download the files.
